Modélisation de base de données

La compétence « modélisation de base de données » s’apparente au fait de pouvoir réfléchir à la conception, créer et administrer des socles de données. Cette compétence donne la possibilité à l’utilisateur de pouvoir bénéficier d’un stockage de ses données tout en ayant un schéma bien défini permettant une bonne organisation. Le choix d’une base de données est rudimentaire pour pérenniser son projet. Il existe différents types de base de données, les plus connues sont les bases de données relationnelles et les bases de données orientées documents (NOSQL). Il y a plusieurs points qui les distinguent.

Base de données relationnelles :

  • Base de données utilisée principalement pour enregistrer et modifier des données structurées
  • Données facilement exploitables via des requêtes SQL
  • La gestion des droits d’accès est facilement applicable

Base de données orientées documents :

  • Utilisée pour manipuler des données semi-structurées et non-structurées
  • Le type de données à implémenter ne nécessite pas d’avoir de type connu
  • Vitesse d’écriture rapide
  • Haute scalabilité

La base de données est un élément important dans des projets professionnels. Elle permet aux entreprises de centraliser et partager leurs informations. En plus de ça, elle donne la possibilité aux utilisateurs d’accéder aux données simultanément et rapidement. Nous pouvons prendre comme exemple Facebook qui stocke les informations des utilisateurs dans une base de données ce qui va permettre un accès aux données utilisateurs rapides et fiables.

Mise en oeuvre

Lors de mon stage à AgenceMD, j’ai eu pour mission de gérer la création et la réflexion d’un socle de données.

AgenceMD avait pour but de créer une base de données regroupant toutes les informations concernant les sociétés ainsi que des données sur ses collaborateurs. L’enjeu était de réfléchir au schéma à adopter pour accueillir au mieux les données. Cette base de données devait ensuite fournir les informations pour plusieurs applications.

La difficulté de la mission était de construire tout une base de données avec des tables connectés entre elles. La base devait contenir un total de quatre tables, La table « contact » (contient toutes les informations liées à un contact), la table « mobile », la table « email », la table « société » (contient les sociétés) et la table « fixe ».

Pour ce faire j’ai décidé d’y aller par étapes pour faciliter la compréhension et avoir une idée claire de mon objectif.

J’ai tout d’abord modélisé les différentes tables sur MySQL Workbench, un logiciel de gestion et d’administration de base de données. Cet outil donne la possibilité de créer un modèle conceptuel des données (MCD) ce qui me permet de me rendre compte des différentes propriétés de mes futures tables ainsi que de leur lien (Foreign Key).


Exemple d’un schéma

Une fois le MCD validé par mon DBA, je me suis alors consacré à créer les tables dans la base de données. Suite à ça j’ai pu remplir les différentes tables avec leurs données correspondantes via la requête suivante : LOAD DATA INFILE 'data.txt' INTO TABLE canal_email;

Une fois les données chargées, les applications ont commencé à consumer les données des tables mais un problème est vite survenu, la lenteur. En effet les tables comportaient énormément de données et ce qui causait des lenteurs sur les applications, elles mettaient beaucoup de temps pour recevoir toutes les données.

C’est alors que j’ai décidé de mettre en place des « index ». Les index permettent de simplifier et d’accélérer les opérations de recherche, de tri ou de jointure entre les tables. Dans mon cas pour simplifier la vitesse de réception des données j’ai mis en place des index destinés aux colonnes de jointure.

Suite à ça, les applications n’avaient plus de problème de lenteur et le schéma de base de données a été validé par le DBA.

Maitrise

Je pense savoir maîtriser la plupart des aspects de la base de données pour ce qui est de la réflexion, conception et mise en place de schéma. Le milieu professionnel dans lequel je me trouvais lors de mon stage m’a été plus que formateur puisqu’il fallait récupérer des informations de l’INSEE et des fichiers de différents partenaires de la société AgenceMD pour ensuite créer des schémas de base de données en conséquence.

En plus de ça j’ai mis en place des procédures complexes permettant l’alimentation en données d’applications.

Le fait de pratiquer régulièrement grâce aux tâches qui m’étaient confiées, m’a permis de monter en compétence rapidement.

Lors de mon cursus scolaire j’ai appris les rudiments de la base de données. J’ai pu mettre en place des socles de données de plus en plus complexes durant les multiples projets réalisés. Une fois arrivé dans le monde professionnel je me suis rendu compte de l’importance des bases de données. Elle est en effet la pièce maîtresse d’une bonne application puisque sans une base de données fiable nous ne pouvons pas avoir une application pérenne.

Lors de mes différentes modélisations de base de données durant mon stage j’ai dû faire face à plusieurs problèmes, notamment sur la relation entre les différentes tables de ma base de données créées pour les diverses applications d’ AgenceMD. Pour pallier à ça j’ai eu la chance d’avoir le soutien d’un expert en base de données qui a pu m’apprendre les bonnes pratiques à adopter pour réaliser un bon socle de données.

Avec l’expérience que j’ai pu accumuler, je conseille aux débutants dans les bases de données de pratiquer le plus souvent possible. Cela va aider à comprendre le fonctionnement d’une base de données. En plus de ça, ils vont acquérir une nouvelle façon d’appréhender un projet, c’est-à-dire, réfléchir à la conception d’un projet suivant le schéma qu’ils voudront définir en base de données. Avec cette réflexion ils auront moins de problèmes liés aux stockages de données ainsi qu’aux relations entre les différentes tables.
