Git

Git est un logiciel de gestion de versions gratuit et open source. Créé par Linus Torvalds en 2005, il est utilisé par plus de 15 millions de personnes. Git aide à analyser plus facilement les modifications et les contributions apportées à du code collaboratif. Cela permet au développeur de garder ses fichiers à jours et d’y accéder n’importe où de façon sécurisée.

La notion de « branche » est importante avec Git. Une branche peut être traduite par une copie du projet pour développer et tester de nouvelles fonctionnalités sans impacter le projet de base. Cette branche peut ensuite être « merge » (fusionné) avec la branche principale « Main » du projet pour y apporter ces modifications.


Exemple de gestion de branche

Git se différencie de la concurrence de par ces nombreuses caractéristiques :

  • Git utilise un modèle de données qui garantit l’intégrité cryptographique de tout ce qui est présent dans un dépôt dans le but de les sécuriser
  • Git propose au développeur de pouvoir formater les modifications et les faire réviser avant de les appliquer réellement. Cela permet de garder un contrôle sur les branches
  • Toutes les actions que Git peut réaliser comme les commits, créer des branches ou faire des merges sont optimisées. Ce qui rend Git très performant par rapport aux autres alternatives

Aujourd’hui de nombreuses entreprises utilisent Git pour stocker et gérer leurs projets. On peut citer CGI où les projets sont stockés et gérés sur Git. Cela permet de centraliser les projets pour que les développeurs puissent reprendre des projets facilement.

Mise en oeuvre

Lors de mon premier semestre, j'ai été initié à Git dans le cadre du projet Logicode, une expérience qui s'est avérée être un tournant dans ma manière de gérer le développement collaboratif. Au début du projet, mon équipe et moi avions opté pour une méthode rudimentaire de partage de fichiers via une clé USB, ce qui s'est rapidement révélé inefficace et chronophage. La prise de conscience de cette lacune nous a conduits à découvrir Git, qui a transformé notre approche en nous permettant de centraliser et de mettre à jour dynamiquement le code source de l'application.

L'adoption de Git a été cruciale pour la réussite du projet Logicode(projet informatique réalisé au deuxième semestre à l'ESIEA), nous permettant de respecter nos délais malgré un démarrage tardif. Bien que mon utilisation de Git à ce stade fût basique, ne profitant pas encore pleinement de ses fonctionnalités avancées comme la gestion des branches, cette première expérience a posé les fondations de ma compétence future avec cet outil.

Dans les projets suivants, j'ai pris l'initiative de démarrer directement avec Git, approfondissant ma compréhension et mon usage de ses capacités, notamment en créant des branches pour chaque nouvelle fonctionnalité. Cette pratique a amélioré significativement l'efficacité et la gestion des versions dans le développement.

Plus récemment, dans le projet GECO, l'utilisation de Git a pris une nouvelle dimension. Alors que je travaillais sur le développement de nouvelles fonctionnalités, la nécessité de retirer un élément complexe de l'application un test d'éligibilité sur les numéros de compte clients s'est présentée. La solution est venue sous la forme d'un Git revert, une commande qui nous a permis de revenir à un état antérieur du code sans la fonctionnalité en question. Cette manœuvre a non seulement économisé un temps précieux, mais a également évité de potentiels dysfonctionnements dans l'application.

Cet épisode a démontré la puissance de Git dans la gestion de situations complexes et a renforcé ma conviction que la maîtrise de cet outil est indispensable pour tout développeur soucieux de l'efficacité et de la fiabilité dans le processus de développement de logiciels.

Maitrise

Actuellement, je me considère comme un utilisateur initié de Git. Bien que j'aie eu l'opportunité d'utiliser Git dans tous mes projets, me permettant ainsi d'améliorer progressivement ma pratique, je suis conscient qu'il me reste encore du chemin à parcourir pour atteindre une maîtrise totale. Git est un outil à la fois puissant et complexe, dont l'apprentissage complet exige des années de pratique. De plus, chaque projet a sa propre manière d'implémenter Git, ce qui ajoute à la complexité de son apprentissage.

La maîtrise de Git est devenue une compétence cruciale dans le domaine de l'ingénierie logicielle, essentielle pour la gestion efficace du code source et des versions d'un projet. Sa maîtrise est fortement valorisée dans le secteur professionnel, de nombreuses entreprises recherchant activement cette aptitude.

Git a une place importante dans le profil d’expert en ingénierie puisqu’il permet de savoir stocker le code source d’un projet et de gérer ses versions. Beaucoup d’entreprises recherchent cette compétence aujourd’hui.

Fort de mon expérience actuelle, je recommande vivement aux débutants de se familiariser avec Git en exploitant des ressources telles que les vidéos YouTube. La chaîne Graven - Développement, par exemple, propose des explications claires sur le fonctionnement de Git dans un contexte de projet. En complément, le jeu Oh My Git offre une approche ludique pour apprendre à utiliser les commandes Git. Ces ressources rendent l'apprentissage de Git plus accessible et engageant, ce qui est particulièrement bénéfique pour ceux qui débutent dans ce domaine.

Évolution compétence

L'utilisation quotidienne de Git dans mes projets actuels est une pratique régulière pour moi, mais je suis conscient que je suis toujours en phase d'apprentissage. Face à la diversité des commandes disponibles dans Git, je me retrouve souvent à consulter la documentation pour exécuter des commandes moins familières. Mon objectif est de me perfectionner en apprenant une plus grande variété de commandes Git, ce qui, je le sais, contribuera à accélérer mon processus de développement.

Pour atteindre cet objectif, je prévois de m'engager dans un processus d'auto-formation ciblé sur Git et ses aspects les plus complexes. Je compte notamment suivre des tutoriels vidéo sur YouTube, et je suis particulièrement intéressé par les contenus proposés par la chaine Grafikart. Ces ressources spécialisées offrent un apprentissage approfondi et pratique de Git, me permettant ainsi d'élargir mon répertoire de compétences et d'améliorer mon efficacité en tant que développeur.
