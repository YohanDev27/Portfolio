Git

Git est un logiciel de gestion de versions gratuit et open source. Créé par Linus Torvalds en 2005, il est utilisé par plus de 15 millions de personnes. Git aide à analyser plus facilement les modifications et les contributions apportées à du code collaboratif. Cela permet au développeur de garder ses fichiers à jours et d’y accéder n’importe où de façon sécurisée.

La notion de « branche » est importante avec Git. Une branche peut être traduite par une copie du projet pour développer et tester de nouvelles fonctionnalités sans impacter le projet de base. Cette branche peut ensuite être « merge » (fusionné) avec la branche principale « Main » du projet pour y apporter ces modifications.


Exemple de gestion de branche

Git se différencie de la concurrence de par ces nombreuses caractéristiques :

  • Git utilise un modèle de données qui garantit l’intégrité cryptographique de tout ce qui est présent dans un dépôt dans le but de les sécuriser
  • Git propose au développeur de pouvoir formater les modifications et les faire réviser avant de les appliquer réellement. Cela permet de garder un contrôle sur les branches
  • Toutes les actions que Git peut réaliser comme les commits, créer des branches ou faire des merges sont optimisées. Ce qui rend Git très performant par rapport aux autres alternatives

Aujourd’hui de nombreuses entreprises utilisent Git pour stocker et gérer leurs projets. On peut citer Capgemini et plus particulièrement sur son pôle Aéronautique ou les projets sont stockés et gérés sur Git. Cela permet de centraliser les projets pour que les développeurs puissent reprendre des projets facilement.

Mise en oeuvre

J’ai eu l’occasion d’utiliser Git pour la première fois lors de mon premier semestre avec le projet KeepClean à la suite d’un retard occasionné durant le développement. Avec mes camarades et moi-même nous avions commencé à développer le jeu KeepClean sans utiliser Git. C’est-à-dire qu’après l’ajout de nouvelles fonctionnalités, nous nous échangions les données sur une clé USB pour garder le projet à jour. Nous nous sommes rendus compte assez rapidement qu’il nous fallait une alternative pour stocker nos données de projet puisque nous perdions énormément de temps. C’est à ce moment que nous avons remarqué l’existence de Git. Il nous a permis de centraliser le code source de l’application et d’ajouter du nouveau code dynamiquement.

L’utilisation de Git sur ce projet nous a permis de rendre le projet à temps sans problème de version.

Sur ce premier projet, je n’ai pas eu l’opportunité d’utiliser pleinement les capacités de Git. Je l’ai utilisé sans savoir qu’il existait un principe de branche. C’est pourquoi, pour le projet E-pop, j’ai démarré le projet directement avec Git. Cela m’a permis d’utiliser plus en profondeur Git en y incorporant des branches pour chaque nouvelle fonctionnalité.

Plus récemment, j’ai utilisé Git pour le Comptage BTOB avec l’équipe informatique. Je devais développer de nouvelles fonctionnalités pour l’application. Après chaque développement, je faisais un « push » (envoyer le code) sur la branche de développement. Un jour le client m’a demandé à ce qu’on retire une grosse fonctionnalité de l’application, le comptage par contact. Sauf que l’on ne pouvait pas simplement retirer le code en le supprimant puisque cela impactait de nombreuses parties de l’application. Pour ce faire, nous avons réalisé un « git revert ». C’est une commande qui nous a permis de revenir à un moment clé de notre branche de développement sans la fonctionnalité du comptage par contact.

Cela nous a fait gagner beaucoup de temps sur notre développement et nous a permis de ne pas causer des problèmes dans l’application en enlevant une fonctionnalité importante.

Maitrise

Pour l’utilisation de Git, j’estime être initié. Il est vrai que j’ai pu utiliser Git sur tous mes projets. Cela m’a permis de me perfectionner sur son utilisation. Mais il est toutefois important de noter qu’il me reste une marge de progression afin de maîtriser totalement cet outil. C’est un outil puissant et complexe qui nécessite des années d’utilisation pour pouvoir le maîtriser. En plus de ça, chaque projet a sa façon d’utiliser Git. Ce qui rend son apprentissage plus difficile.

Grâce à ce langage, j’ai pu me rendre compte du potentiel qu’avaient les bibliothèques pour simplifier le travail du développeur. En plus de ça, PHP est un langage facile à apprendre de par ses nombreuses documentations que l’on peut retrouver en ligne.

Git a une place importante dans le profil d’expert en ingénierie puisqu’il permet de savoir stocker le code source d’un projet et de gérer ses versions. Beaucoup d’entreprises recherchent cette compétence aujourd’hui.

Avec l’expérience que j’ai aujourd’hui, je conseille à tout débutant de se renseigner par le biais de vidéo YouTube avec la vidéo de Graven - Développement qui explique très bien comment Git fonctionne sur un projet. En plus de ça il existe un jeu s’intitulant « Oh My Git » qui permet d’apprendre à utiliser des commandes Git tout en s’amusant.

Évolution compétence

Aujourd’hui, j’utilise quotidiennement Git pour mes projets. Mais je suis toujours en apprentissage et je dois donc souvent regarder la documentation si je veux effectuer des commandes que je n’ai pas l’habitude d’utiliser puisqu’il en existe une multitude. J’aimerais ainsi m’améliorer en apprenant le plus possible de commandes Git pour m’aider à améliorer ma vitesse de développement.

Pour ce faire, je compte m’auto-former sur l’utilisation de Git ainsi que de ses commandes complexes en suivant des vidéos YouTube spécialisées dans l’utilisation de Git de la chaine Grafikart
